## Summary

交付物（执行阶段将落地为文档与模板）：

- 方法论文档（流程 + 模块说明 + 示例）
- 可直接复制的 Prompt 模板（提炼 / 改写 / 验证）
- 评分表（rubric，0-10 分维度）与验收流程

## Goals & Success Criteria

### 目标

- 从账号的历史内容中提炼：
  - 行业/领域：该账号长期覆盖的主题地图、概念体系、术语口径、典型案例库
  - 理念/偏好/立场：价值观、反对项、取舍原则、风险偏好、对受众的默认假设
  - 心智模型：3-7 个可运行的模型（含适用边界与触发条件）
  - 决策启发式：可被新情境触发的规则列表（每条带证据与反例）
  - 文风 DNA：可量化、可描述、可复刻（但不过度“模仿秀”）
- 用双重验证机制验证“提炼结果”与“文风复刻”的可信度与一致性。

### 验收（执行完成后应可按此验收）

- 任意抽取 5 篇账号原文：
  - 能在提炼产物中找到对应证据链接与归因（“证据 → 结论”的可追溯性）
  - 关键模型至少满足 2/3 的三重验证（跨域复现/生成力/排他性）
- 任意抽取 3 个新问题做“模型推断”：
  - 推断结果与账号既有观点不冲突，且能给出“依据模型的推理链”
- 文风复刻：
  - 量化指标在容忍区间内（句长、语气、结构、修辞密度等）
  - 人工对照清单通过（读者能明显感知“像这个账号”）

## Implementation Plan (Methodology Design)

### 1) 输入准备：内容语料与切分规范

1. 语料收集与归一化
   - 来源：文章/视频稿/直播整理等
   - 统一格式：`标题、发布日期、平台、正文、原始链接、备注`
2. 切分粒度
   - “证据最小单元”：段落（或 2-4 句的语义块）
   - “主题单元”：一篇内容的核心观点与论证结构
3. 建立验证集（holdout）
   - 按时间或主题分层抽样留出 20% 内容作为验证集，不参与提炼，只用于检验推断与风格复刻

### 2) 领域提炼：行业地图与概念体系

目标：从“内容表层主题”抽象出“领域结构”，形成可导航的知识地图。

产物：

- 领域地图（3 层）：领域 → 子议题 → 典型问题
- 概念词表：术语定义、口径、同义词/反义词、常用类比
- 案例库：该账号常引用的案例类型与来源偏好

方法步骤：

1. 主题聚类：为每篇内容标注 1-3 个主题标签（可从开放编码开始，再收敛到固定 taxonomy）
2. 概念抽取：为每个语义块抽取“概念/术语/指标/框架名”
3. 口径对齐：为高频概念写“账号口径定义”，并收集“反例与误用提醒”
4. 领域边界：记录“账号不覆盖/不讨论/明确反对”的范围（与受众定位相关）

验证：

- 跨内容一致性：同一概念在不同内容中的定义不冲突；若冲突，按“演化/场景/张力”标注
- 可检索性：读者能从“典型问题”反查到原文证据

### 3) 观点谱系：理念/偏好/立场/取舍原则

目标：将“说了什么”升级为“长期相信什么、讨厌什么、怎么取舍”。

产物（建议以卡片化表达）：

- 核心信念（Beliefs）：不可妥协的原则与世界观假设
- 偏好矩阵（Preferences）：效率 vs 完整、增长 vs 风险、短期 vs 长期等取舍
- 反对清单（Anti-beliefs）：明确反对的做法/观点/路径
- 受众假设：对读者能力、资源、目标的隐含假设

方法步骤：

1. 观点提取：每个语义块归纳为“主张句（可被反驳的陈述）”
2. 归因层级：区分“事实判断 / 价值判断 / 策略判断”
3. 取舍识别：遇到“但是/然而/取决于/权衡”时，抽取其权衡维度与阈值条件
4. 反对项：识别否定句、反讽、警告类表达，形成“账号红线”

验证：

- 反例覆盖：每条“偏好/原则”至少有 1 个反例或边界条件
- 可推断：给定一个新情境，能用偏好矩阵解释其可能选择

### 4) 心智模型提炼：从观点到“可运行的模型”

目标：提炼 3-7 个“可运行”的模型（不是原话集合），用于生成对新问题的推断。

模型卡片结构（建议固定字段）：

- 模型名（账号口吻命名）
- 一句话定义
- 核心变量与关系（输入/中介/输出）
- 触发条件（何时使用）
- 推断规则（If/Then 或步骤）
- 典型证据（至少 3 条语义块，来自不同主题/场景优先）
- 适用边界与失效条件
- 反例/常见误用

三重验证（从参考框架迁移）：

1. 跨主题复现：同一模型能在至少 2 个不同子议题出现（或同一变量关系在不同问题上反复出现）
2. 有生成力：能对“验证集中新问题”给出可解释的推断
3. 有排他性：体现账号独特视角（与行业共识/大众观点可区分）

降级规则：

- 只通过 1 重验证：记为“决策启发式”
- 0 重验证：记为“情境性表达”，不纳入长期模型

### 5) 文风 DNA 提取：句式指纹 + 结构习惯 + 词汇口径

目标：把“读起来像”拆成可测量与可执行的写作约束。

提取维度（核心）

1. 句式指纹（统计）
   - 平均句长、短句比例、疑问句比例、祈使句比例
   - 转折频率（但是/然而/不过）、因果连接（因此/所以/因为）
   - 类比/比喻密度、例子密度（每千字）
   - 第一人称/第二人称使用率
   - 确定性语气（一定/显然）vs 谨慎语气（可能/也许）
2. 结构习惯（模板）
   - 结论先行 vs 铺垫型
   - 段落节奏：观点 → 解释 → 例子 → 反例 → 总结
   - 标题与小标题风格：问题式/命题式/数字清单式
3. 词汇与修辞
   - 高频词与关键词（账号“常用概念”）
   - 禁忌词（几乎不用/刻意避免）
   - 口癖（高频短语，要求“适度使用”）
   - 情绪色彩：冷静/讽刺/鼓励/警告等

风格标签（描述性）

- 正式↔口语、抽象↔具体、谨慎↔断言、学术↔通俗、长句↔短句、数据驱动↔叙事驱动、结论先行↔铺垫型

### 6) 双重验证：量化阈值 + 人工对照清单

A. 量化验证（先过线）

- 从账号语料中抽样 20 段建立“基线区间”（均值与分位数范围）
- 对“复刻文本”计算同类指标，判断是否落在容忍区间
- 关键指标建议分为：
  - 必须命中：句长区间、结构模板命中率、关键词覆盖率、禁忌词违例=0
  - 可容忍偏离：类比密度、第一人称比例、确定性语气比例（设上下限）

B. 人工对照（再确认）

- 盲测：给评审者 A/B 两段（原文 vs 复刻/提炼文），让其判断“哪个更像账号”
- 对照清单（示例维度）：
  - 是否沿用账号的概念口径（同词同义）
  - 是否遵守账号的结构节奏（先结论/先故事等）
  - 是否出现账号不会用的措辞（禁忌词）
  - 语气是否一致（断言/谨慎、是否对读者“下指令”）
  - 是否有“账号典型的反例/提醒方式”

### 7) Prompt 模板体系：把方法论变成可执行操作

将输出组织为“可复制 Prompt”，每个 Prompt 明确：

- 输入变量（语料、主题、目标受众、输出长度、保留的口癖强度等）
- 输出格式（卡片字段、表格字段、JSON/YAML 任选其一）
- 自检步骤（先生成，再按 rubric 自评并修订）

模板分组：

- 语料归一化与切分
- 观点抽取与归因
- 领域地图与概念口径
- 心智模型卡片生成（含三重验证）
- 文风 DNA 统计与总结
- 文风改写（保持事实不变、调整语气与结构）
- 双重验证（量化检查清单 + 人工盲测脚本）

### 8) 评分表（Rubric）：0-5 分可打分验收

评分维度建议（每项 0-5 分，给扣分点与满分定义）：

- 可追溯性：每条结论是否能回链到证据
- 领域结构：地图是否清晰、无明显空泛词、可导航
- 模型可运行性：是否能对新问题推断且给出推理链
- 边界与反例：是否明确何时失效、避免过度泛化
- 文风一致性（量化）：关键指标命中情况
- 文风一致性（人工）：盲测通过率与主观一致性
