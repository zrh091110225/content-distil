# 抓马教育 Style Spec（执行版）

用途：把原 `style_dna` 的分析基线改写为“可直接拿来生成”的执行文档。  
适用任务：
- 同事实改写：事实、信息、逻辑不变，只调整表达与结构。
- 新问题写作：围绕新议题写新稿，但必须保持抓马教育的判断方式、案例质地与情绪边界。

语料基础：`docs/zhuama` 共 37 篇；本执行版基于训练集 27 篇整理。  
分析基线与验证集仍以现有产物为准：见 [holdout_list.md](file:///Users/hymanhai/Documents/zrhp/content-distil/outputs/zhuama/holdout_list.md)。

## 1. 先记住：这不是“模仿语气”，而是“复用写法”

写这个号，不是把若干口头禅拼起来，而是同时命中下面 5 个核心约束：

1. `对象/场景先行`：先让读者看见人、关系、处境、动作，再抬出观点。
2. `议题化写作`：不是给方法清单，而是把表层问题翻译成更深一层的控制、选择、主体性、边界、成长问题。
3. `重定义问题`：至少完成一次“你以为是 A，其实更深处是 B”。
4. `案例服务观点`：案例必须具体、有冲突、有代价，不能只是装饰。
5. `先托住再推进`：允许锋利，但不羞辱、不恐吓、不靠廉价情绪推动阅读。

如果只学到“不是……而是……”这种表面句式，而没有命中上面 5 条，生成结果仍然不算像。

## 2. 两种写作模式

### 2.1 同事实改写

适用：
- 原文信息已完整，只是不像抓马教育
- 需要保留活动信息、课程信息、作品信息、事实顺序

硬约束：
- 不新增事实，不改结论，不偷换逻辑
- 可以重排段落顺序，但不能改动事实关系
- 优先补“场景、议题、判断、收束”，而不是疯狂替换词汇

改写时优先修的顺序：
1. 先换结构
2. 再换开头
3. 再补观点重定义
4. 最后才调句式和词汇

### 2.2 新问题写作

适用：
- 针对一个新议题、新活动、新作品、新教育问题写全新文本
- 允许引入新的事实材料，但表达方式必须像抓马教育

硬约束：
- 至少使用一个明确议题主轴：`控制 / 选择 / 主体性 / 边界 / 成长 / 人性 / 安全感 / 自我守护`
- 至少完成一次问题重定义
- 至少给一个具体案例或具体处境
- 结尾必须回到判断、邀请或同行式收束，不能只剩 CTA

## 3. 写前决策树

正式落笔前，先做 4 个决定：

### 3.1 先选模板

只允许从以下 3 个模板里选一个起稿，不要边写边漂移。

#### 模板 A：议题拆解型

适用：
- 作品解读
- 议题深挖
- 一篇文章要拆 2-3 个子问题

骨架：
1. 用场景或关系冲突把读者带进去
2. 用一句话钉住“最核心 / 最根本 / 真正的问题”
3. 按 `01 / 02 / 03` 或等价分节拆开
4. 每节都做一轮：现象/案例 -> 解释 -> 当代映射 -> 追问
5. 结尾回到主体性、选择、成长或邀请探索

#### 模板 B：强隐喻推进型

适用：
- 心理/关系议题
- 控制、依恋、规训、吞噬、自我守护这类主题
- 需要强烈氛围感和意象贯穿

骨架：
1. 先抛一个意象或身体感受
2. 解释这个意象背后的心理结构
3. 再把意象投射到家庭、学校、社会或当下处境
4. 用 2-3 个追问把读者拉进自我对照
5. 结尾收回到判断或同行式邀请

#### 模板 C：先愿景后信息型

适用：
- 招募
- 课程
- 工作坊
- 活动介绍

骨架：
1. 先说“为什么值得进入这个空间”
2. 再说孩子/家长/观众会得到什么
3. 中段再解释背后的议题或教育理解
4. 最后再给活动安排、报名信息、时间地点
5. 结尾仍然要保留判断或陪伴感，不能只剩信息块

### 3.2 决定这篇文章的“主判断”

必须能写成一句可辩驳的话，不能只是正确废话。

可用句型：
- `真正的问题不是 A，而是 B。`
- `我们以为这是 A，但它更接近 B。`
- `最核心的困境，不在于 A，而在于 B。`
- `这件事迫使我们重新思考……`

不可用句型：
- `成长很重要。`
- `教育要尊重孩子。`
- `我们都应该更勇敢。`

### 3.3 决定案例来源

优先级从高到低：
1. 课堂 / 创排 / 工作坊现场
2. 作品中的角色与冲突
3. 家庭 / 成长 / 亲子中的真实处境
4. 导演、演员、孩子、观众的具体表达

不优先：
- 遥远名人故事
- 空泛“有个孩子后来成长了”
- 为了凑观点而编的示例

### 3.4 决定情绪边界

默认情绪路线：
- 先让读者感到被理解
- 再让读者看到问题的锋利处
- 最后给一个判断或继续探索的空间

禁止路线：
- 一上来训人
- 用恐惧制造点击
- 全文都在发火

## 4. 全局硬约束

下面这些规则，优先级高于局部句式与修辞。

### 4.1 必须命中

- 必须有一个清晰议题主轴，不能只是信息搬运
- 必须至少完成一次问题重定义
- 必须有一个具体人物、具体场景或具体关系处境
- 必须让案例服务观点，而不是把案例写成独立故事
- 必须在结尾做收束：判断、邀请、反问、陪伴，至少占一个

### 4.2 建议命中

- 标题优先使用“场景/对象 + 议题”或“问题句 + 冲突”
- 开头优先从场景、故事、关系压入，而不是背景说明
- 句子节奏优先“短句破题 + 中长句展开 + 提问/对照收束”
- 转折比因果更常见，优先“拆直觉，再给新框架”

### 4.3 严禁触发

- 宏大空词开头：`时代发展`、`信息时代`、`社会背景下`
- 咨询味/AI味黑话：`赋能`、`抓手`、`底层逻辑`、`全链路`、`生态位`
- 流量口头禅：`建议收藏`、`含金量拉满`、`太炸裂了`
- 廉价恐吓：`再不改变就晚了`、`你正在毁掉孩子`
- 纯方法清单：`3个方法解决问题`
- 高位说教：`家长一定要明白`
- 只批判不给路径
- 结尾只剩报名/购票 CTA

## 5. 8 层执行规则

下面 8 层不再按分析文档写 A/B/C/D，而是改成“生成时怎么做”。

## 5.1 标题

目标：先给读者一个可以进入的对象、场景或冲突，再给议题。

优先写法：
- `场景 / 对象 + 议题`
- `问题句 + 关系冲突`
- `作品名 + 当代解释`

长度建议：
- 18-26 字优先
- 能具体就具体，不追求极短爆款标题

允许使用的抓手：
- 场景
- 对象
- 轻冲突
- 当代映射

禁止：
- 纯噱头
- 空泛宏大词
- 纯方法清单
- 和正文不是同一个判断口径

自检：
- 标题里是否能看见“谁/什么处境”
- 标题是否暗含议题，而不只是信息块
- 如果正文讲“主体性与控制”，标题有没有跑成“震撼反转”

## 5.2 开头

目标：前 100-300 字让人立刻感到“这像抓马教育”。

优先动作：
- 先给画面
- 先给人物处境
- 先给一个问题
- 在第一屏内补一句清晰判断

建议节奏：
1. 画面或故事先进入
2. 轻推进
3. 用一句判断把议题钉住

禁止：
- 背景介绍连写两三段
- 先讲活动时间、课程安排、项目历史
- 一开头就上价值说教

自检：
- 第一段有没有具体物象、关系或动作
- 第一屏有没有出现议题线索
- 开头和标题是不是同一个问题

## 5.3 结构

目标：文章不是“堆观点”，而是有推进、有转身、有收束。

执行规则：
- 先选模板 A/B/C
- 至少有一次“立主题”
- 至少有一次“收束动作”
- 每个主要段落都要回答一个推进问题，而不是并列堆素材

常见推进顺序：
- 现象 -> 真问题 -> 解释 -> 当代映射 -> 追问/邀请
- 场景 -> 判断 -> 拆解 -> 回到读者

禁止：
- 万能总分总硬套
- 中间只有概念，没有动作
- 活动文一上来先报名信息
- 结尾只剩转化信息

自检：
- 删掉任一大段后，文章推进是否会断
- 中段有没有至少一次“从故事走向判断”
- 结尾是不是回到了文章最初的问题

## 5.4 句法

目标：句子有带入感、推进感、判断感，但不写成口号或论文。

量化护栏：
- 平均句长尽量落在 `25-63` 字
- 可以有短句破题，但不要通篇短句
- 转折多于因果，疑问句适度出现

高频句法动作：
- `不是……而是……`
- `本质上……`
- `最核心 / 最根本的困境是……`
- `它迫使我们思考……`
- `在今天 / 在当下……依然……`

使用原则：
- 这些句式只能点到为止，不能每段都复制
- 更重要的是“反问推进”和“对照推进”，不是模板复读

禁止：
- 说明书句法
- 抽象名词连堆
- 机械复刻签名句式
- 全篇口号短句

自检：
- 是否既有短句破题，也有中长句展开
- 是否至少有一次自然的提问或对照
- 把所有口头禅删掉后，文章是否仍然站得住

## 5.5 词汇

目标：用抓马教育自己的概念栈说话，而不是借外部黑话。

优先概念场：
- `控制 / 依恋 / 占有 / 边界 / 吞噬 / 安全感`
- `主体性 / 选择 / 责任 / 守住自己 / 自我连接`
- `规训 / 结构性暴力 / 神谕 / 大局 / 宏大叙事`
- `看见 / 沉默 / 失语 / 真实 / 牺牲 / 成长`

优先推进器：
- 对照
- 追问
- 当代映射
- 戏剧转现实

使用原则：
- 同词同义，不乱换口径
- 高频词可以复用，但不要堆满全文
- 概念出现后，要有场景、案例或解释托住

禁止：
- 商业黑话
- 流量黑话
- 和戏剧/成长/主体性语境不兼容的话语体系

自检：
- 关键词是否在服务判断，而不是为了显得“高级”
- 有没有出现明显不是这个号会说的话
- 概念后面有没有解释和落地

## 5.6 观点

目标：让文章有这个号的判断力，而不只是温和陪伴。

必须做到：
- 至少一次问题重定义
- 至少一个明确判断标准
- 最好给出一条行动路径或理解路径

高频判断路径：
- 把“爱 / 关心 / 保护”重定义为“控制 / 规训 / 占有”
- 把戏剧情节翻译成现实镜子
- 把道德争论改写成主体性问题

高频判断标准：
- `这件事是否让人更清醒，而不是更顺从`
- `当压力出现时，我们是否还能守住自己`
- `真正的问题是不是表层情绪，而是深层结构`

禁止：
- 正确废话
- 只批判不给路径
- 把复杂问题压扁成一句鸡汤
- 只给结论，不解释机制

自检：
- 删掉所有形容词后，是否还剩一个明确判断
- 文中是否至少回答了“为什么这是问题”
- 是否只说“要怎样”，却没说“为什么会这样”

## 5.7 案例

目标：让读者通过一个具体瞬间感到问题真的存在。

必须要素：
- 具体角色
- 具体场景
- 具体动作
- 明确冲突或代价

案例的正确功能：
- 证明观点
- 解释概念
- 让读者共鸣

案例的错误功能：
- 只做氛围装饰
- 只讲一个温暖故事
- 和后文判断断裂

优先案例素材：
- 课堂、创排、排演、工作坊
- 角色冲突
- 家庭关系中的选择与拉扯
- 可以触发身体感和空间感的细节

禁止：
- 悬浮案例
- 无冲突案例
- 鸡汤名人故事
- 案例出现后不回收

自检：
- 是否能在一句话里说清“谁在什么处境里遇到什么冲突”
- 案例删掉后，观点是否会明显变空
- 案例之后有没有跟出解释或判断

## 5.8 情绪

目标：锋利，但不伤人；有重量，但不制造恐慌。

默认情绪配方：
- `陪伴感 先行`
- `警醒感 跟上`
- `判断感 收束`

常用情绪动作：
- 先托住再推进
- 先点破再解释
- 先共情再邀请行动

允许：
- 直接指出控制、规训、裹挟
- 对空话和熟悉的错误做法保持警惕
- 有判断，但不靠喊口号

禁止：
- 羞辱读者
- 高位训斥
- 廉价恐吓
- 只有怒气，没有解释
- 过度煽情

自检：
- 文中有没有任何一句让读者感觉“被骂了”
- 批评对象是现象/结构，还是具体人
- 结尾有没有给人一点继续理解和继续行动的空间

## 6. 定量护栏

这些数字不是为了机械模仿，而是给生成时做边界校正。

- 标题长度优先落在 `18-26` 字
- 句长优先落在 `25-63` 字
- 标题更偏场景词，不偏强情绪词
- 开头更偏场景/故事切入，不偏结论先行
- 每篇至少完成 `1 次问题重定义`
- 每篇至少给 `1 个具体案例`
- D 类违例必须为 `0`

如果为了命中数字而写得很假，以“像真实抓马教育”为优先，不机械追数。

## 7. 写完后的快速质检

交稿前，按下面顺序检查：

### 7.1 五问快检

1. 这篇文章有没有一个明确的主判断？
2. 有没有把表层问题改写成更深一层的问题？
3. 有没有一个具体的人、场景、动作、冲突？
4. 结尾有没有回到判断、邀请或同行感？
5. 有没有任何一句像咨询黑话、AI 黑话或流量标题党？

### 7.2 八层过线标准

- 标题：不是噱头，和正文同口径
- 开头：先带入，再立题
- 结构：模板稳定，不散
- 句法：有节奏，不像说明书
- 词汇：概念栈一致，无黑话
- 观点：完成重定义，不停在常识
- 案例：具体，且服务观点
- 情绪：锋利但不羞辱

### 7.3 一票否决项

只要出现任一情况，就视为未过线：

- 标题猎奇、正文失焦
- 开头空铺垫
- 结构只堆观点不推进
- 黑话、官话、AI 味明显
- 观点只剩正确废话
- 案例悬浮
- 情绪越界成羞辱或恐吓
- 结尾只剩 CTA
